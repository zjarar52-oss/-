# Voice Calendar Assistant

语音日历助手产品访问：https://voice-calendar-assistant.netlify.app/

---

1. 项目背景（Project Background）

在移动互联网高度普及的今天，日历早已成为每个人管理时间的重要工具。然而，当我们重新审视大学生日常生活中的日程管理行为时，会发现现有产品并没有真正解决他们的核心问题。

大学生的时间往往是碎片化的。课程、考试、社团活动、竞赛报名、实习面试、聚会安排、纪念日提醒等事务交织在一起，形成大量需要记录与提醒的时间节点。但现有日历产品大多沿袭传统的输入逻辑：打开应用、点击日期、填写标题、设置时间、保存事件。对于临时想到的事项而言，这种操作链路依然过长。

与此同时，许多重要信息并不属于严格意义上的"待办事项"。朋友生日、获奖经历、旅行纪念、毕业时刻等内容，同样与时间有关，却往往散落在社交软件、备忘录和相册之中，缺乏统一的时间管理入口。

因此，本项目尝试提出一种新的解决方案：以自然语言交互为核心，将"记录时间"这件事从手动输入转变为直接表达。用户不再需要思考如何填写表单，而只需要像和朋友说话一样描述自己的安排。Voice Calendar Assistant 由此诞生。

---

2. 项目简介（Project Introduction）

Voice Calendar Assistant 是一款面向大学生群体设计的轻量化语音日历工具。

项目以"自然语言记录日程"为核心理念，通过语音交互降低时间管理工具的使用门槛，让用户能够在碎片化场景下快速完成日程创建与管理。

与传统办公日历不同，本项目并不追求复杂的团队协作能力，而是聚焦个人时间管理场景，希望帮助用户以更轻松的方式完成日程记录、提醒管理与个人记忆留存。

本项目同时探索了"未来事务管理"与"过去记忆存档"的融合模式，将待办事项与重要纪念事件统一纳入时间轴视角进行展示。

---

3. 产品设计思路（Design Philosophy）

产品整体围绕两个核心场景展开：

3.1 当下与未来

这一部分承担传统日历工具的职责。用户可以记录未来安排、查看近期事项，并获得对应提醒。

相比传统输入模式，Voice Calendar Assistant 希望用户能够直接使用自然语言进行表达，例如：

> 明天下午三点开组会
> 周五晚上七点和同学聚餐
> 下个月十号提交论文初稿

系统将自动识别时间与事件信息，并生成对应日程。这种设计的核心目标并不是替代日历，而是降低使用日历的心理成本。

3.2 过去与记忆

时间管理并不仅仅意味着安排未来。很多时候，人们更希望记录已经发生的重要时刻。因此产品增加了记忆留存模块。

用户可以记录：
- 获奖经历
- 毕业纪念
- 旅行回忆
- 聚会记录
- 人生重要节点

这些内容将以时间轴形式保留在对应日期中，使日历不仅记录未来，也承载过去。

4. 核心功能说明（Core Features）

4.1 语音日程录入

用户通过语音或文本输入自然语言内容。系统自动提取：
- 日期
- 时间
- 事件内容

例如：

| 用户输入 | 解析结果 |
|----------|----------|
| 明天下午三点开组会 | 生成对应会议日程 |
| 周五晚上七点聚餐 | 生成聚餐安排 |
| 下周二交作业 | 生成作业提醒 |

通过这种方式，用户无需经历复杂表单填写流程即可完成记录。

4.2 日程管理

系统支持基础的日程管理能力。用户可以：
- 新增日程
- 查看日程
- 编辑日程
- 删除日程

所有信息均以日历视图展示。用户能够快速查看每日安排与未来计划。

4.3 提醒机制

对于重要事项，用户可自主开启提醒功能。提醒方式采用轻量化设计。支持：
- 提前10分钟提醒
- 提前30分钟提醒
- 提前1小时提醒

避免过度通知带来的干扰问题。

4.4 隐私保护模式

考虑到大学生常处于教室、宿舍、图书馆等公共环境中，项目增加隐私控制设计。

用户可选择：

| 模式 | 展示内容 |
|--------|--------|
| 完整模式 | 显示具体日程内容 |
| 隐私模式 | 仅显示待办数量 |

在保证效率的同时保护个人隐私。

4.5 生日管理

除了普通日程管理之外，项目还增加了生日记录场景。用户能够：
- 记录亲友生日
- 查看本月生日名单
- 查询未来生日安排

帮助用户更方便地管理重要关系节点。

4.6 高光时刻记录

高光时刻模块用于记录个人成长过程中的重要事件。例如：
- 获奖
- 升学
- 实习Offer
- 毕业
- 旅行纪念

这些内容将与日期关联，形成个人成长时间轴。

8. 产品创新点（Innovation）

与传统日历产品相比，本项目主要有三个特点：

第一，强调自然语言交互。用户不需要学习复杂操作流程，而是直接使用口语表达完成记录。

第二，将未来日程与过去记忆统一纳入时间管理体系。日历不仅记录待办事项，也成为个人成长记录工具。

第三，坚持轻量化设计原则。不引入复杂团队协作机制，不增加冗余功能，而是专注解决个人时间管理问题。
